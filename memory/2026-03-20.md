## Admin Role JWT Issue
- **Issue:** PapaBear's Coinbase wallet login was returning `role=user` instead of `role=admin`
- **Root cause:** Duplicate wallet records for same address with different casing (checksummed vs lowercase)
  - `0xb9CEF66FD6A8728c04311217756AB080c7DA4e32` → mapped to admin user
  - `0xb9cef66fd6a8728c04311217756ab080c7da4e32` → mapped to different user with `role=user`
- **Fix:** Manually consolidated duplicate wallets, remapped canonical row to admin user
- **Result:** JWT now correctly returns `role=admin` after re-login

## Wallet Uniqueness Migration (0010)
Created migration to prevent future case-variant duplicates for non-Solana chains.

### Iteration 1: Failed on existing duplicates
- Migration tried to create unique index but existing duplicates blocked it
- Error: `Key (chain, lower(address::text))=(ethereum, 0x3dbda69a6f8c60777f070b1379db69f28c736fa9) is duplicated`

### Iteration 2: Failed on min(uuid)
- Added deduplication logic using `min(id)` to pick canonical wallet
- Error: `function min(uuid) does not exist`
- PostgreSQL doesn't support min/max aggregates on UUID type

### Iteration 3: Success
- Replaced `min(id)` with `DISTINCT ON (chain, lower(address)) ORDER BY id::text`
- Dedupe steps:
  1. Identify canonical wallet per `(chain, lower(address))` using deterministic text sort of UUID
  2. Repoint `wallet_indexing_jobs` and `wallet_holdings_snapshots` to canonical row
  3. Delete duplicate wallet rows
  4. Create `wallets_chain_address_ci_unique` index on `(chain, lower(address)) WHERE chain <> 'solana'`
- **Commits:** `82699e4`, `716d8ff`, `4c3db4a` on `dev`

## Technical Debt Notes
- Auth service normalization happens in `findOrCreateUser()` but wallet linking uses separate `normalizeAndValidateAddress()` in `me.service.ts`
- Both paths should use same normalization to prevent future case-variant issues
- Consider extracting address normalization to shared utility

## Lessons
- Always dedupe before creating uniqueness constraints in migrations
- PostgreSQL UUID type doesn't support min/max aggregates; use `::text` cast or `DISTINCT ON` instead
- Case-variant addresses are a common footgun in EVM wallet systems; normalize early and enforce at DB level
