### System Overview

Built a 4-phase spam detection and management system for NEXUS collections:

**Phase 1: Database Schema** (`0015_add_spam_management.sql`)
- Added spam fields to collections table: `is_spam`, `spam_score`, `spam_reason`, `spam_detected_at`, `spam_detected_by`
- Created `spam_reports` table for user reports
- Created `spam_allowlist` table for verified legitimate collections
- Enum types: `spam_report_type`, `spam_detected_by`

**Phase 2: Automatic Detection**
- Integrated Alchemy API `spamClassifications` field during holder indexing
- Auto-flags collections with `spam_score >= 80`
- Stores lower scores for manual review
- Detection source tracked (`alchemy`, `manual`, `helius`, `user_report`)

**Phase 3: Admin UI**
- Spam stat card showing total flagged collections
- "Show spam" toggle (default: hide)
- "🚫 Spam Only" filter option
- Spam badges: 🚫 SPAM (red) for confirmed, ⚠️ Score: X (yellow) for suspicious
- Action buttons: "🚫 Mark Spam" / "✓ Not Spam"

**Phase 4: Bulk Processing**
- `SpamCheckerService`: Retroactively checks all existing collections
- `POST /admin/collections/bulk-check-spam` endpoint
- "🔍 Bulk Check Spam" button in admin UI
- Sequential processing with progress logging
- Returns summary: `{ checked, flagged, errors }`

### Technical Details

**API Integration:**
- Alchemy: `getContractMetadata` endpoint for spam classifications
- EVM chains only: Ethereum, Base, Polygon, Abstract, ApeChain
- Solana: No spam detection available in Helius API

**Threshold Logic:**
- Score >= 80: Auto-flag as spam
- Score 1-79: Store for review, don't auto-flag
- Score 0: Clean

**Database Relations:**
- Removed circular dependency by not using `.references()` in spam tables
- Foreign keys enforced by migration SQL only
- Prevents `chainEnum is not a function` error

**Admin Actions:**
- Mark spam: Sets `is_spam=true`, `spam_score=100`, `spam_detected_by='manual'`
- Mark not spam: Clears spam fields, adds to `spam_allowlist`
- Allowlist prevents future auto-detection

### Lessons Learned

**Circular Dependencies:**
- Drizzle schema imports can create runtime errors
- Use plain `uuid()` fields without `.references()` when circular
- Foreign keys enforced by migration SQL still work

**Spam Detection Strategy:**
- Automatic high-confidence flagging (score >= 80)
- Manual review for suspicious (score 1-79)
- Allowlist for false positives
- Multiple detection sources tracked

**API Rate Limits:**
- Sequential processing avoids 429 errors
- Progress logging every N items
- Error tolerance per-item
- ~5 minutes for 500 collections
